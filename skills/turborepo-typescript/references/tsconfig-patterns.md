# TypeScript Config Patterns Reference

> Authority: [turborepo.dev/docs/guides/tools/typescript](https://turbo.build/repo/docs/guides/tools/typescript) and [turborepo.com/blog/you-might-not-need-typescript-project-references](https://turborepo.com/blog/you-might-not-need-typescript-project-references) (v2.x)

## `@repo/typescript-config` structure

```
tooling/typescript-config/
├── package.json
├── base.json           # Extended by all other configs
├── nextjs.json         # For Next.js apps
├── react-library.json  # For React component packages (JIT)
├── nestjs.json         # For NestJS services
└── node.json           # For Node CLI tools and compiled packages
```

```json
// tooling/typescript-config/package.json
{
  "name": "@repo/typescript-config",
  "version": "0.0.0",
  "private": true,
  "files": ["*.json"],
  "license": "MIT"
}
```

## `base.json`

Shared across all packages. Applies strict safety options.

```jsonc
// tooling/typescript-config/base.json
{
  "$schema": "https://json.schemastore.org/tsconfig",
  "display": "Base",
  "compilerOptions": {
    "target": "es2022",
    "lib": ["es2022"],
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "strict": true,
    "isolatedModules": true,
    "moduleDetection": "force",
    "noUncheckedIndexedAccess": true,
    "noImplicitOverride": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true
  }
}
```

**Key options explained:**

| Option | Why |
|--------|-----|
| `strict` | Enables all strict type checks |
| `isolatedModules` | Required for transpile-only tools (esbuild, tsup, swc) — catches re-export-type issues |
| `moduleDetection: force` | Treats every file as a module; prevents ambient type pollution |
| `noUncheckedIndexedAccess` | Adds `undefined` to array/object index access results |
| `skipLibCheck` | Skips type-checking of `.d.ts` files; prevents noise from dep type bugs |
| `module: NodeNext` | Required for correct ESM/CJS interop in Node; pairs with `moduleResolution: NodeNext` |

## `nextjs.json`

Extends `base.json` for Next.js apps:

```jsonc
// tooling/typescript-config/nextjs.json
{
  "$schema": "https://json.schemastore.org/tsconfig",
  "display": "Next.js",
  "extends": "./base.json",
  "compilerOptions": {
    "target": "es2017",
    "lib": ["dom", "dom.iterable", "esnext"],
    "module": "esnext",
    "moduleResolution": "bundler",
    "allowJs": true,
    "jsx": "preserve",
    "noEmit": true,
    "resolveJsonModule": true,
    "incremental": true,
    "plugins": [{ "name": "next" }]
  }
}
```

Usage:
```jsonc
// apps/web/tsconfig.json
{
  "extends": "@repo/typescript-config/nextjs.json",
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx", ".next/types/**/*.ts"],
  "exclude": ["node_modules"]
}
```

`next-env.d.ts` is auto-generated by Next.js. Add it to `.gitignore` and always include it in tsconfig.

## `react-library.json`

For JIT React component packages:

```jsonc
// tooling/typescript-config/react-library.json
{
  "$schema": "https://json.schemastore.org/tsconfig",
  "display": "React Library",
  "extends": "./base.json",
  "compilerOptions": {
    "lib": ["dom", "dom.iterable", "esnext"],
    "module": "esnext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "noEmit": true
  }
}
```

## `nestjs.json`

For NestJS services (compiled output, CommonJS):

```jsonc
// tooling/typescript-config/nestjs.json
{
  "$schema": "https://json.schemastore.org/tsconfig",
  "display": "NestJS",
  "extends": "./base.json",
  "compilerOptions": {
    "module": "CommonJS",
    "moduleResolution": "Node",
    "target": "es2021",
    "lib": ["es2021"],
    "outDir": "dist",
    "declaration": true,
    "declarationMap": true,
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true
  }
}
```

`experimentalDecorators` and `emitDecoratorMetadata` are required for NestJS's DI system.

## `node.json`

For non-NestJS Node tools and compiled shared packages:

```jsonc
// tooling/typescript-config/node.json
{
  "$schema": "https://json.schemastore.org/tsconfig",
  "display": "Node",
  "extends": "./base.json",
  "compilerOptions": {
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "outDir": "dist",
    "declaration": true,
    "declarationMap": true
  }
}
```

## When `composite: true` is correct

`composite: true` is only required when using TypeScript Project References. The Turborepo team recommends against project references — Turborepo's own caching layer handles incremental builds more reliably.

WRONG — enabling composite without project references:
```jsonc
{
  "compilerOptions": {
    "composite": true,     // requires declarationDir/rootDir, adds unnecessary constraints
    "declaration": true,
    "incremental": true
  }
}
```

CORRECT — composite only with project references (not recommended):
```jsonc
{
  "compilerOptions": {
    "composite": true,
    "declaration": true,
    "declarationDir": "dist/types",
    "rootDir": "src"
  },
  "references": [
    { "path": "../../packages/ui" }
  ]
}
```

For most repos: use Compiled strategy packages instead of project references. They give the same build-order guarantees with simpler config.
